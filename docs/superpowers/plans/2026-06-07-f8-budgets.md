# F8 — Budgets Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Per-category monthly budgets for Cóndor. A user sets an optional monthly budget (in their base currency) per category; the app shows spent-vs-budget progress on the **Inicio** bars (subtle under-budget fill, accessible over-budget tint + icon + text) and as a spent/budget + pct line in each **Categorías** row. Budgets persist through the existing `Repository` seam in all three implementations (Local, Supabase, Syncing) and flow through the F4 outbox like every other entity. Deleting a custom category cleans up its budget. Everything is mobile-first, i18n (es/en), and TDD.

**Architecture:** Adds a `Budget` domain type (`{ id, categoryId, amountBase, period: 'monthly', createdAt, updatedAt }`) + zod schema, mirroring the exact patterns in `lib/domain/types.ts` / `lib/domain/schemas.ts`. Extends the `Repository` interface (`lib/data/repository.ts`) with `listBudgets()/upsertBudget(b)/deleteBudget(id)` — implemented in `LocalStorageRepository` (key `condor:budgets`), `SupabaseRepository` (the `budgets` table created in F3, mapped via `lib/data/supabase-mappers.ts`), and `SyncingRepository` (the `'budget'` entity already reserved in F4's `OutboxEntity` union — F8 fills the `default:` no-op branches in `findRemote`/`writeRemote`/`writeLocal`/`applyRemoteDelete`). A **pure** `budgetProgress(expenses, budgets, month)` selector in `lib/domain/selectors.ts` does the spent-vs-budget math (TDD, the quick win). Zustand store gains `budgets` state + `setBudget`/`deleteBudget` actions that persist via the repo (mirrors `addCategory`/`updateCategory`). UI: the budget field lives in `NewCategorySheet` (Categorías add/edit sheet); progress renders in `RankedBars` (Inicio) and `CategoriasPage` rows. Invariant preserved: components only touch the store + selectors, never `localStorage`/`supabase-js`/`fetch`. This is F8 of the revised Phase-2 design (§3, table `budgets` from F3, outbox slot from F4).

**Tech Stack:** TypeScript · Next 16 App Router (`'use client'` pages) · Zustand vanilla store + `useStore` hook · Zod (`z.object`, `.parse`) · next-intl (`messages/es.json`, `messages/en.json`) · Vitest (jsdom, `globals`, `@/` alias → repo root) + `@testing-library/react` + `userEvent` · Supabase JS v2 (`{ data, error }`, `.upsert({ onConflict })`) for the cloud repo · lucide-react icons · Tailwind (tokens: `bg-surface`, `text-muted-txt`, `text-danger`, `font-money`). Package manager: **pnpm** (never npm/yarn). No new runtime dependencies.

## Task 2 — Quick win: `budgetProgress` pure selector (TDD)

**Files:** `lib/domain/selectors.ts`, `tests/unit/selectors.test.ts` (append)

The heart of F8. Pure function, no I/O — the fastest high-value thing to land. For the given month, joins each budget to the spent total of its category and emits progress. Categories *without* a budget produce **no row** (so the UI stays quiet). `spentBase` sums `baseAmount` (null skipped, same rule as every other selector).

**Contract:**
```
budgetProgress(expenses, budgets, month) => Array<{
  categoryId: string;
  spentBase: number;     // sum of baseAmount for that category in the month
  budgetBase: number;    // the budget's amountBase
  pct: number;           // spentBase / budgetBase * 100; 0 when budgetBase === 0
  over: boolean;         // spentBase > budgetBase
}>
```
Rules: one row per budget (only the latest budget per category — dedupe by categoryId keeping the most recent `updatedAt`); `pct` is **not** clamped (can exceed 100 so the UI can show "180%"); `over` is `spent > budget` (strict — exactly 100% is NOT over); when `budgetBase === 0`, `pct = 0` and `over = spentBase > 0`. Output sorted by `categoryId` asc for determinism.

```ts
// ---------------------------------------------------------------------------
// budgetProgress
// ---------------------------------------------------------------------------

export interface BudgetProgressRow {
  categoryId: string;
  spentBase: number;   // sum of baseAmount (null skipped) for the category in month
  budgetBase: number;  // the budget cap
  pct: number;         // spentBase / budgetBase * 100; 0 when budgetBase === 0; NOT clamped
  over: boolean;       // spentBase > budgetBase (strict; exactly 100% is not over)
}

/**
 * Per-category budget progress for the given month.
 * - One row per BUDGETED category (categories without a budget produce no row).
 * - When a category has multiple budgets, the newest by updatedAt wins.
 * - spentBase sums baseAmount; null baseAmounts (unconverted FX) are skipped.
 * - pct is unclamped so the UI can render e.g. "180%"; 0 when budgetBase === 0.
 * - over is strict (> budget), so exactly 100% is on-budget, not over.
 * - Rows are sorted by categoryId asc for deterministic rendering.
 */
export function budgetProgress(
  expenses: Expense[],
  budgets: Budget[],
  month: string,
): BudgetProgressRow[] {
  if (budgets.length === 0) return [];

  // Dedupe budgets by categoryId, keeping the newest updatedAt.
  const latest = new Map<string, Budget>();
  for (const b of budgets) {
    const prev = latest.get(b.categoryId);
    if (!prev || b.updatedAt > prev.updatedAt) latest.set(b.categoryId, b);
  }

  // Sum spent per category for the month (null baseAmounts skipped).
  const inMonth = expensesInMonth(expenses, month);
  const spent = new Map<string, number>();
  for (const e of inMonth) {
    if (e.baseAmount === null) continue;
    spent.set(e.categoryId, (spent.get(e.categoryId) ?? 0) + e.baseAmount);
  }

  const rows: BudgetProgressRow[] = [];
  for (const [categoryId, budget] of latest) {
    const spentBase = spent.get(categoryId) ?? 0;
    const budgetBase = budget.amountBase;
    const pct = budgetBase > 0 ? (spentBase / budgetBase) * 100 : 0;
    rows.push({ categoryId, spentBase, budgetBase, pct, over: spentBase > budgetBase });
  }

  rows.sort((a, b) => a.categoryId.localeCompare(b.categoryId));
  return rows;
}
```
