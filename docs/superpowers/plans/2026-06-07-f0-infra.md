# F0 — Infra Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convert Cóndor from a static export (`output: 'export'`) into a server-capable Next.js app deployed on Vercel, wired to the already-provisioned Supabase project (`svgphkbtspqgsliqbsfx`), with env vars in place locally and in Vercel, the install dependencies for cloud features (`@supabase/supabase-js`, `@supabase/ssr`), a CI gate on PRs, and a verified preview deploy — **without breaking the existing hand-rolled PWA (manifest + `public/sw.js`)**. No app behavior changes yet; CRUD keeps running against the existing local repo.

**Architecture:** This is the F0 row of the Phase-2 revised design (`docs/superpowers/specs/2026-06-07-condor-phase2-revised-design.md`). It only prepares infra; later features (F1 Auth, F3 Cloud repo, F5 Server FX) consume it. The PWA stays as-is: `public/sw.js` is hand-rolled and registered client-side by `components/common/ServiceWorkerRegister.tsx`; `public/manifest.webmanifest` is a static asset. Dropping static export does NOT touch routing (i18n is message-based via `lib/i18n/messages.ts`, not URL-prefixed) and does NOT touch the SW (still served from `public/`). The only server surface F0 introduces is the build target itself — no API routes yet.

**Tech Stack:** Next.js 16.2.7 (server runtime — dropping `output: 'export'`) · `@supabase/supabase-js` + `@supabase/ssr` (installed, not yet wired) · Supabase project `svgphkbtspqgsliqbsfx` · Vercel project `condor` (account `dux14`, prod https://condor-ecru-gamma.vercel.app/) · GitHub repo `dux14/condor-expense-tracker` · GitHub Actions CI · existing: Zustand, Zod, next-intl, Tailwind v4, hand-rolled `public/sw.js`. **pnpm throughout — never npm/yarn.**

## Risks / things to watch

- **Frozen lockfile in CI:** if `pnpm-lock.yaml` is out of sync after `pnpm add`, CI's `--frozen-lockfile` fails. Section 6.4 catches this locally; always commit the updated lockfile.
- **PWA navigation caching:** `public/sw.js` does network-first for navigations and falls back to `/`. The server build still serves the same routes, so the SW keeps working — but bump `CACHE_VER` (currently `condor-v2`) in a later feature if cached static export HTML ever lingers on devices. F0 does not need to bump it.
- **Image optimization now active on Vercel** (was `unoptimized` under static export). If any `next/image` usage relied on the unoptimized path, watch the preview for broken images; none expected since the app shipped static before.
- **Vercel `vercel deploy` vs Git-connected deploys:** if the `condor` project is Git-connected, pushing the branch would also trigger a preview. Section 7 uses an explicit CLI deploy so it works before any push.
